## Aktivitas 3: Pembuatan Script `sync_prodi_dosen.py` (Sinkronisasi Prodi + Dosen + Mahasiswa)

**File:** `utils/pddikti/sync_prodi_dosen.py`
**Referensi:** `utils/pddikti/docs/sinkronisasi_prodi_dosen_mahasiswa.md`

### Alur Script

1. Cari URL profil PT via halaman pencarian PDDikti
2. Baca semester aktif dari dropdown "Data Pelaporan Tahunan"
3. Baca tabel prodi (11 kolom termasuk dosen, mahasiswa, rasio)
4. Capture URL detail tiap prodi dengan cara **klik nama prodi** (link bersifat dinamis/hover), catat URL navigasi, lalu kembali ke halaman PT
5. Upsert `universities_datadosen` dari data tabel PT
6. Buka halaman detail tiap prodi aktif:
   - Ekstrak profil: telepon, email, website, tanggal berdiri, SK, deskripsi
   - Update `universities_programstudi`
7. Klik tab Tenaga Pendidik → baca tabel dosen, ikuti pagination → upsert `universities_profildosen`
8. Klik tab Mahasiswa → baca 3 baris semester → upsert `universities_datamahasiswa`

### Debugging yang Dilakukan

| Masalah | Solusi |
|---|---|
| Link detail prodi tidak ada di HTML | Klik cell nama prodi, capture `driver.current_url` |
| `MoveTargetOutOfBoundsException` | `scrollIntoView` sebelum klik |
| ENV_PATH salah (3 level ke atas) | `parent.parent.parent / ".env"` |
| Nama tabel `universities_profiledosen` | Nama sebenarnya: `universities_profildosen` |
| `_read_table_rows` ambil tabel Riwayat | Cari tabel berdasarkan header "NIDN" |
| `_read_mahasiswa_table` tidak menemukan data | Cari tabel berdasarkan header "Semester" |
| Kontak tanpa label di HTML | Regex pattern matching pada `body.text` |
| Teks "-" pada informasi umum | `clean_block()` abaikan nilai "-" |

## Aktivitas 4: Pembuatan Script `sync_detail_dosen.py` (Sinkronisasi Detail Individu Dosen)

**File:** `utils/pddikti/sync_detail_dosen.py`
**Referensi:** `utils/pddikti/docs/sinkronisasi_detail_dosen.md`
**Bergantung pada:** `utils/pddikti/scrape_pddikti_detaildosen.py` (sudah ada sebelumnya)

### Tujuan

Melengkapi data profil individu dosen yang sebelumnya hanya berisi nama dan NIDN
(dari sync_prodi_dosen), dengan menambahkan:
- **Profil lengkap:** jabatan fungsional, jenis kelamin, pendidikan tertinggi, ikatan kerja, status, NUPTK, URL pencarian
- **Riwayat pendidikan:** jenjang, perguruan tinggi asal, gelar, tahun lulus, flag luar negeri

### Alur Script

1. Baca daftar dosen dari `universities_profildosen` berdasarkan `kode_pt`
2. Buka browser Firefox headless sekali untuk semua dosen
3. Untuk tiap dosen:
   - Cari di PDDikti: `search/[nama dosen] [nama PT]` → klik tab Dosen → ambil link "Lihat Detail"
   - Buka halaman detail dosen
   - Ekstrak profil via regex dari `body.text`
   - Ekstrak riwayat pendidikan dari tabel (filter: hanya tabel dengan header "Gelar" atau "Jenjang")
4. UPDATE `universities_profildosen`
5. DELETE lama + INSERT baru `universities_riwayatpendidikandosen`
6. Commit per dosen, jeda 3 detik antar request

### Mapping Data

| Field PDDikti | Tabel DB | Field DB |
|---|---|---|
| Jabatan Fungsional | universities_profildosen | jabatan_fungsional |
| Pendidikan Terakhir | universities_profildosen | pendidikan_tertinggi |
| Status Ikatan Kerja | universities_profildosen | ikatan_kerja |
| Jenis Kelamin | universities_profildosen | jenis_kelamin |
| Status Aktivitas | universities_profildosen | status |
| NUPTK | universities_profildosen | nuptk |
| URL pencarian | universities_profildosen | url_pencarian |
| Perguruan Tinggi (riwayat) | universities_riwayatpendidikandosen | perguruan_tinggi_asal |
| Gelar Akademik | universities_riwayatpendidikandosen | gelar |
| Jenjang | universities_riwayatpendidikandosen | jenjang |
| Tahun | universities_riwayatpendidikandosen | tahun_lulus |
| (auto-detect) | universities_riwayatpendidikandosen | is_luar_negeri |

## Aktivitas 5: Fix Login 401 di Frontend + Backend

### Masalah

Error `POST /api/auth/login/ 401` muncul secara intermittent.

### Penyebab & Fix

**Penyebab 1 — `SessionAuthentication` di production settings:**
DRF mencoba autentikasi session jika ada cookie lama di browser. Session auth memaksa CSRF check → request login tanpa `X-CSRFToken` ditolak.

Fix: hapus `SessionAuthentication` dari `DEFAULT_AUTHENTICATION_CLASSES` di `ptma/settings/production.py`.

**Penyebab 2 — `AuthInterceptor` mengirim token ke semua endpoint:**
Token lama yang expired di `localStorage` dikirim ke `/api/auth/login/` → server tolak dengan 401.

Fix: tambahkan `publicPaths` di `auth.interceptor.ts`, skip Authorization header untuk endpoint publik.

**Penyebab 3 (root cause sebenarnya) — DRF authentication berjalan sebelum permission check:**
Meskipun `@permission_classes([AllowAny])` terpasang, `TokenAuthentication` tetap berjalan lebih dulu. Token expired → 401 sebelum view dieksekusi.

Fix: tambahkan `@authentication_classes([])` pada semua view publik:
- `login_view`, `mfa_verify`, `register_view`, `forgot_password`, `reset_password`
